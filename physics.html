Approach

The point of this page is not to look technical. It is to make the technical reasoning easier to revisit.

  • Start with a physical question. If the page cannot name the phenomenon clearly, the rest turns into decoration.
  • Keep the first honest failure. That is usually the point where the model starts teaching something real.
  • Save the revealing plot. One good graph usually explains more than several polished screenshots.
Best metadata to keep
Project type

Next move
